Микроразметка FAQ в журнале

Микроразметка необходима для корректного отображения блока в поисковой выдачи. Среди возможных вариантов рекомендуеться исполтзовать тип разметки json-ld.

Для работы нужно использовать этот шаблон:

<script type="application/ld+json">
    {
      "@context": "https://schema.org",
      "@type": "FAQPage",
      "mainEntity": [{
        "@type": "Question",
        "name": "Каким компаниям подходит автоворонка продаж?",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "Практически любому бизнесу B2B и B2C. Особенно если у него много клиентов или сложный процесс продажи. Например, в сфере информационных продуктов — онлайн-курсы и сервисы; здравоохранения — телемедицина, медицинские центры и лаборатории; туризм — гостиницы, базы отдыха и турагентства. Автоворонки также подходят для розницы, где возможны повторные и комплексные продажи — магазины, супермаркеты, интернет-магазины."
        }
      }, {
        "@type": "Question",
        "name": "Чем автоворонка лучше обычной воронки продаж?",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "Преимущества автоворонки: выполняет задачи автоматически, освобождая работников от монотонной и рутинной работы; меньше затраты на зарплату менеджеров; обрабатывает заявки круглосуточно; более точно выявляет потребности аудитории и на их основе создает персонализированные предложения."
        }
      }, {
        "@type": "Question",
        "name": "От чего зависит эффективность автоворонки?",
        "acceptedAnswer": {
          "@type": "Answer",
          "text": "От множества факторов. Например, качества трафика, оффера, качества самой автоворонки, оптимизации под мобильные устройства, контента, слаженности работы между сайтом и CRM."
        }
      }]
    }
    </script>

В массиве json в ключе лучше не использовать кавычки. потому что тогда Google может не понять происходящее. Для корректной работы ссылок внутри ключей лучше использовать одинарные кавычки.

“text”: “Компания ‘Электрон’ выпустила новый релиз”;

Для проверки корректности можно использовать интерфейс Google https://search.google.com/test/rich-results?hl=ru

Important

Интерфейс Google может не показывать размеченные области, несмотря на то, что код присутствует на сайте. Это происходит из-за того, что запрашиваемая страница еще не была проиндексирвоана. Чтобы базово проверить коррнетность микроразметки нужно использовать вкладку “Код”
